# Lista Withdraw 取款逻辑说明

## 概述
本文档详细说明 `lista_withdraw.py` 的取款金额计算逻辑。

## 输入参数
- **max_withdraw**: 合约中最大可取出金额（由合约的 `maxWithdraw()` 方法返回）
- **config_amount**: 用户配置的取出金额（通过 `--amount` 参数设置）

## 取款规则

### 情况1: 剩余可取金额 < 配置金额的20%
**条件**: `max_withdraw < config_amount * 0.2`

**操作**: ❌ 跳过取款，等待下次检查

**原因**: 可取金额太少，不值得取出

**示例**:
- 配置金额 = 10
- 最大可取 = 0.5 (< 2.0)
- 结果: 跳过

---

### 情况2: 配置金额的20% <= 剩余可取金额 < 配置金额
**条件**: `config_amount * 0.2 <= max_withdraw < config_amount`

**操作**: 
1. 计算 `int_amount = int(max_withdraw)` (向下取整)
2. 如果 `int_amount < 1`，则跳过取款
3. 否则取出 `int_amount`

**原因**: 剩余金额不足配置金额，但超过20%阈值，可以取出整数部分（最小为1）

**示例**:
- 配置金额 = 10, 最大可取 = 2.0 → 取出 int(2.0) = **2**
- 配置金额 = 10, 最大可取 = 3.5 → 取出 int(3.5) = **3**
- 配置金额 = 10, 最大可取 = 9.9 → 取出 int(9.9) = **9**
- 配置金额 = 1, 最大可取 = 0.5 → int(0.5) = 0 < 1 → **跳过**
- 配置金额 = 1, 最大可取 = 0.99 → int(0.99) = 0 < 1 → **跳过**

---

### 情况3: 剩余可取金额 >= 配置金额
**条件**: `max_withdraw >= config_amount`

**操作**: 取出 `max(max_withdraw * 0.2, config_amount)`

**原因**: 
- 当可取金额充足时，优先取出较大的金额
- 要么是配置金额，要么是可取金额的20%（取两者的最大值）
- 这样可以最大化资金利用效率

**示例**:
- 配置金额 = 10, 最大可取 = 10 → 取出 max(2, 10) = **10**
- 配置金额 = 10, 最大可取 = 15 → 取出 max(3, 10) = **10**
- 配置金额 = 10, 最大可取 = 50 → 取出 max(10, 10) = **10**
- 配置金额 = 10, 最大可取 = 100 → 取出 max(20, 10) = **20**

---

## 特殊规则
1. **最小取出金额**: 1（任何情况下取出金额 < 1 都会跳过）
2. **向下取整**: 在情况2中使用 `int()` 向下取整
3. **不超过最大可取**: 计算出的金额永远不会超过 `max_withdraw`

---

## 决策流程图

```
开始
  ↓
获取 max_withdraw
  ↓
max_withdraw < config_amount * 0.2?
  ├─ 是 → 跳过取款
  └─ 否 ↓
     max_withdraw < config_amount?
       ├─ 是 → int_amount = int(max_withdraw)
       │        ↓
       │      int_amount < 1?
       │        ├─ 是 → 跳过取款
       │        └─ 否 → 取出 int_amount
       │
       └─ 否 → 取出 max(max_withdraw * 0.2, config_amount)
```

---

## 动态间隔功能
- **初始状态**: 使用配置的间隔（如 `--interval 60`）
- **首次成功后**: 自动切换到快速模式（1秒间隔）
- **快速模式持续**: 每次成功后继续使用1秒间隔
- **首次失败后**: 自动恢复原始间隔

这样可以在资金充足时快速连续取出，在资金不足时降低检查频率节省资源。

---

## 设计目标
1. **避免频繁小额交易**: 通过20%阈值过滤太小的金额
2. **最大化资金效率**: 在资金充足时取出较大金额（20%或配置金额）
3. **清理碎片金额**: 在情况2中取出接近配置金额的整数部分
4. **保持最小标准**: 任何取出金额都不能小于1

---

## 命令行使用

```bash
# 每60秒检查一次，配置金额10
python lista_withdraw.py --interval 60 --amount 10

# 每30秒检查一次，配置金额1，关闭音频
python lista_withdraw.py --interval 30 --amount 1 --no-sound

# 只执行一次
python lista_withdraw.py --amount 10 --once
```
